### ⚡ التثبيت والتشغيل

```bash
# 1️⃣  استنساخ المشروع
git clone https://github.com/Top1-Quantum-AI/Top1-Quantum-AI.github.io.git
cd Top1-Quantum-AI.github.io

# 2️⃣  تثبيت التبعيات
npm install

# 3️⃣  تشغيل بيئة التطوير
npm run dev
```

> 🎉 افتح المتصفح على **[http://localhost:3000](http://localhost:3000)**

<details>
<summary><b>🐳 التشغيل عبر Docker</b></summary>

```bash
# بناء وتشغيل كل الخدمات
docker compose up --build

# تشغيل في الخلفية
docker compose up -d
```

</details>

<details>
<summary><b>☸️ النشر على Kubernetes</b></summary>

```bash
# تطبيق ملفات الإعداد
kubectl apply -f k8s-deployment.yaml
kubectl apply -f k8s-monitoring.yaml

# التحقق من حالة النشر
kubectl get pods -n quantum-ai
```

</details>

### 🛠️ أوامر مفيدة

| الأمر | الوصف | متى تستخدمه؟ |
|-------|-------|-------------|
| `npm run dev` | 🔥 خادم التطوير مع HMR | أثناء التطوير |
| `npm run build` | 📦 بناء نسخة الإنتاج | قبل النشر |
| `npm run preview` | 👁️ معاينة نسخة الإنتاج | للتحقق من البناء |
| `npm run lint` | 🔍 فحص الكود بـ ESLint | قبل الـ commit |
| `npm run format` | 💅 تنسيق الكود بـ Prettier | تنسيق تلقائي |
| `npm run type-check` | 🔷 فحص أنواع TypeScript | عند تعديل الأنواع |
| `npm test` | ✅ تشغيل كل الاختبارات | قبل رفع PR |
| `npm run server` | 🖥️ تشغيل الخادم الخلفي | تطوير الـ API |

**🔑 إدارة الأسرار**

- OWASP Dependency Check تلقائي
- صور Docker موقّعة بـ Cosign

</td>

## 📖 التوثيق

### 🌍 متغيرات البيئة

```env
# ─── Frontend (Vite) ────────────────────────────────
VITE_API_URL=http://localhost:3001
VITE_OPENAI_API_KEY=your-openai-key-here

# ─── Backend ────────────────────────────────────────
PORT=3001
MONGODB_URI=mongodb://localhost:27017/quantum-ai
REDIS_URL=redis://localhost:6379
JWT_SECRET=your-super-secret-jwt-key

# ─── Quantum Computing ──────────────────────────────
IBM_QUANTUM_TOKEN=your-ibm-quantum-token
AWS_BRAKET_REGION=us-east-1
```

> 📌 انسخ [`.env.example`](.env.example) إلى `.env` وعدّل القيم حسب بيئتك.

### 🌐 مسارات API

| الطريقة | المسار | الوصف | المصادقة |
|:-------:|--------|-------|:--------:|
| `POST` | `/api/v1/auth/login` | تسجيل الدخول | ❌ |
| `POST` | `/api/v1/auth/register` | إنشاء حساب جديد | ❌ |
| `GET` | `/api/v1/quantum/status` | حالة المعالجات الكمية | ✅ JWT |
| `POST` | `/api/v1/quantum/run` | تشغيل دائرة كمية | ✅ JWT |
| `POST` | `/api/v1/ai/analyze` | تحليل البيانات بالذكاء الاصطناعي | ✅ JWT |
| `GET` | `/api/v1/network/nodes` | حالة عقد الشبكة | ✅ JWT |
| `GET` | `/health` | فحص صحة الخادم (liveness) | ❌ |
| `GET` | `/ready` | جاهزية الخادم (readiness) | ❌ |

## 🤝 المساهمة

نرحب بكل مساهمة — من إصلاح خطأ إملائي إلى ميزة كاملة! 🙌

### خطوات المساهمة

```bash
# 1. Fork ثم استنسخ المشروع
git clone https://github.com/YOUR_USERNAME/Top1-Quantum-AI.github.io.git

# 2. أنشئ فرعاً جديداً باسم وصفي
git checkout -b feat/quantum-circuit-visualizer

# 3. طوّر وتأكد من اجتياز الاختبارات
npm test && npm run lint

# 4. اعتمد التغييرات وفق Conventional Commits
git commit -m 'feat: add quantum circuit visualizer'

# 5. ادفع الفرع وافتح Pull Request
git push origin feat/quantum-circuit-visualizer
```

### 📝 قواعد الالتزام (Conventional Commits)

| البادئة | الاستخدام | مثال |
|---------|----------|------|
| `feat:` | ميزة جديدة | `feat: add qubit topology view` |
| `fix:` | إصلاح خطأ | `fix: correct RTL alignment in dashboard` |
| `docs:` | تحديث التوثيق | `docs: update API reference` |
| `style:` | تنسيق بلا تغيير في المنطق | `style: format with prettier` |
| `refactor:` | إعادة هيكلة الكود | `refactor: extract quantum service` |
| `test:` | إضافة أو تحديث اختبارات | `test: add AI model unit tests` |
| `chore:` | مهام صيانة | `chore: update dependencies` |

> 📖 راجع [CONTRIBUTING.md](CONTRIBUTING.md) للتفاصيل الكاملة.
